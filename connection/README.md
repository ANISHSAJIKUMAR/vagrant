# Connection Folder Guide

This folder contains all connection details generated by the VPN VM.

## End-to-end scripts (from project root)

```bash
cd /Users/anishskumar/vagrant
./start_vpn.sh
```

`start_vpn.sh` starts VM, provisions VPN, creates client profile, and populates this folder.
It also validates required artifacts like `endpoint.txt`, `server.env`, and client config.

```bash
cd /Users/anishskumar/vagrant
./stop_cleanup_vpn.sh
```

`stop_cleanup_vpn.sh` stops/destroys VM and removes generated files not required.

## Files in this folder

- `server.env`: server connection metadata
- `endpoint.txt`: VPN endpoint in `IP_OR_DNS:PORT` format
- `server_public.key`: WireGuard server public key
- `clients/<name>/<name>.conf`: WireGuard client profile
- `clients/<name>/details.env`: client details summary

## What is `endpoint.txt` used for?

`endpoint.txt` contains only one value, for example:

`117.202.181.193:51820`

Use cases:
- Quick check of current public server address and port.
- Copy/paste into manual WireGuard setup screens.
- Verify whether endpoint changed after ISP IP change or reprovision.

If you import `clients/<name>/<name>.conf`, you usually do not need to open `endpoint.txt` because endpoint is already inside that client config.

## Quick connect (mobile)

1. Install the WireGuard app on Android/iPhone.
2. Open `clients/<name>/<name>.conf`.
3. Import it in the WireGuard app (or scan QR from VM script output).
4. Toggle VPN ON.

## Quick connect (desktop/laptop)

1. Install WireGuard.
2. Import `clients/<name>/<name>.conf`.
3. Activate tunnel.

## How to create/update a client profile

From project root:

```bash
cd /Users/anishskumar/vagrant
WG_SERVER_PUBLIC_IP="117.202.181.193" WG_PORT="51820" vagrant up --provider=vmware_desktop
vagrant ssh -c 'sudo /vagrant/scripts/add-client.sh brother 10.8.0.2/32'
```

This refreshes files under `connection/` and `connection/clients/brother/`.

## How to SSH into the VM

```bash
cd /Users/anishskumar/vagrant
vagrant ssh
```

Optional SSH config export:

```bash
vagrant ssh-config > connection/vagrant-ssh-config.txt
ssh -F connection/vagrant-ssh-config.txt default
```

## Important network requirement

Router must forward `UDP 51820` to your host machine LAN IP, and host firewall must allow `UDP 51820`.

## Security

- Share only client profile files in `clients/<name>/`.
- Do not share private keys or VM internal key files.
